# 양계장 VLA 실패 원인 분석 (Failure Analysis)

**과제**: OpenVLA-7B(LoRA fine-tune)가 LIBERO 시뮬레이션에서 IR 열화상으로 렌더된 양계장에서
살아있는(따뜻한) 병아리 사이의 죽은(차가운/파란) 병아리를 골라 바구니에 담는 task.
**증상**: 파인튜닝된 모델이 평가에서 성공률 **0%** (로봇이 정지하거나 표류).
**작성일**: 2026-06-09 (진단 세션 종합)

---

## 0. 한 줄 요약 (TL;DR)

모델은 **학습에 실패한 것이 아니다.** 학습 데이터(in-distribution) 이미지를 넣으면 정답 액션을
**오차 ~5 mm**로 예측한다. 0%의 진짜 원인은 ① **학습/평가 관측(observation) 불일치 3종**(이미지 방향,
thermal 해상도, 장면 분포)이 학습된 정책을 평가 때 무력화시킨 것과, ② **거친(bang-bang) 액션 표현**이
마지막 정밀 파지(grasp)를 학습 불가능하게 만든 것이다. **데이터 양 부족이 아니다.**

---

## 1. 진단 방법론

| 방법 | 목적 |
|---|---|
| In-distribution 예측 테스트 | 학습 이미지를 모델에 직접 입력 → 예측 액션 vs 정답 비교 (모델이 배웠나?) |
| 롤아웃 모션 정량화 | 프레임간 변화량으로 "정지 vs 동작" 판정 |
| 폐루프 계측 | 매 스텝 eef 위치·그리퍼·병아리 위치 추적 |
| 통제 비교 | 평가 셋업 장면 vs 수집분포 장면(랜덤+wander)에서 동일 모델 실행 |
| grasp-Z 정밀 로깅 | 병아리 근처에서 eef Z·병아리 Z·그리퍼 상태 추적 |

---

## 2. 핵심 전제: 모델은 학습에 성공했다

학습 데모의 thermal 이미지를 그대로 모델에 입력해 예측 액션과 정답을 비교:

| step | 정답 위치 액션 | 예측 위치 액션 | 오차 |
|---|---|---|---|
| 131 | [0.500, 0.500, 0.285] | [0.498, 0.498, 0.282] | 0.004 |
| 141 | [0.500, 0.168, 0.184] | [0.498, 0.165, 0.173] | 0.012 |
| 522 | [0.500,-0.500, 0.265] | [0.498,-0.498, 0.278] | 0.014 |

→ 위치 예측 오차 ~5 mm. **정책은 task를 정확히 학습했다.** 따라서 데이터 양(100→200)이나
학습 시간(15k→30k)은 원인이 아니다(아래 §6에서 배제 근거 정리).

---

## 3. 실패 원인 사슬 (4가지)

### 원인 1 — 이미지 좌우 반전 (orientation mismatch) [수정됨]

- **수집**(`scripted_blue_chick.py`): agentview를 `img[::-1]`(세로 뒤집기만)로 저장 → 좌우 정상
- **평가**(`libero_utils.get_libero_image`): `img[::-1, ::-1]`(180° 회전 = 추가 좌우 반전)
- robosuite raw 이미지가 상하 반전돼 있어, `[::-1]`은 올바른 방향, `[::-1,::-1]`은 **좌우 거울상**이 됨.
- **증거**: 학습 이미지를 좌우 반전해 입력하면 모델의 **Y축(좌우) 예측 부호가 뒤집힘**
  (정답 y=+0.5 → 정상 입력 +0.498, 반전 입력 **−0.498**).
- **효과**: 평가 때 로봇이 좌우로 반대로 움직임 → 타깃 도달 실패. 다른 OOD와 겹쳐 사실상 정지.
- **수정**: 평가 관측을 수집과 동일한 `[::-1]`로 통일.

### 원인 2 — thermal 해상도/순서 불일치 [수정됨]

- **학습**: 수집 때 256×256 agentview에 thermal_fx를 baked → 로더가 224로 축소 (thermal@256 → 224)
- **평가**: 224로 먼저 축소한 뒤 thermal_fx 적용 (224 → thermal@224)
- thermal_fx의 blur(σ=1.1)·bloom(σ=8.0)·colorbar는 **픽셀 단위** 연산이라 224 vs 256에서 결과가 다름.
- **증거**: 동일 장면에서 thermal@224 입력은 예측이 약화됨 (Y −0.5 → **−0.06**), thermal@256은 정상(Y −0.5).
- **효과**: 평가 이미지가 미세하게 OOD → 폐루프 표류.
- **수정**: 평가도 `agentview[::-1]` @256 → thermal_fx(colorbar=True) @256 → 224 축소 (수집과 동일 순서).

### 원인 3 — 평가 셋업 OOD (고정 BDDL + 정지 병아리) → 위치추정 8 cm 오차

- **평가**: 고정 BDDL 1개, 살아있는 병아리 **정지** (배회 시스템이 수집 스크립트에만 있고 평가 코드엔 없음).
- **학습**: 매 데모마다 **랜덤 생성** BDDL + 살아있는 병아리 **배회(wander)**.
- **증거 (통제 비교)**:

  | 장면 | eef가 죽은 병아리에 접근한 최소 수평거리 |
  |---|---|
  | 평가 셋업(고정+정지) | 매 에피소드 **~8.3 cm**에서 정지 (10/10, 체계적) |
  | 수집분포(랜덤+wander) | **1 mm / 3 mm / 2.2 cm / 5 cm** (사실상 병아리 위) |

- **효과**: 평가 장면이 학습 분포 밖이라 공간 위치추정이 8 cm 빗나감.
- **상태**: 평가를 학습 분포(랜덤 장면 + wander)로 맞추면 1 mm까지 정확 → 해결 가능(아직 기본 적용 안 함).

### 원인 4 (핵심·미해결) — bang-bang 거친 액션 → 정밀 파지 실패

- 스크립트 컨트롤러 게인이 과대(`GAIN_XY=8`, `GAIN_Z=6`) → 타깃이 6 cm만 멀어도 액션이 ±0.5로 포화.
  - 전체 스텝의 **48.7%가 위치축 ±0.5 포화**, 위치 델타 **중앙값 0.492**, 분포가 양극단(≈9% 정지 / ≈49% 최대).
- 이 거친 표현으론 병아리 바로 위에서의 **미세 조정**과 **정확한 그리퍼 타이밍**이 불가능.
- **증거 (grasp-Z 정밀 진단, 수집분포 장면)**:
  - eef가 XY를 **1 mm**까지 맞추고 Z도 잡기 높이까지 하강 (z갭 +0.166 → **+0.008**).
  - 그러나 그리퍼가 **50/50 스텝 모두 열림(1.00) — 단 한 번도 안 닫힘.**
  - 게다가 XY가 완벽한 순간엔 Z가 높고, Z가 맞으면 XY가 1.5 cm 드리프트 → **동시 정렬 실패.**
- **효과**: 병아리 위까지 정확히 가서도 지속 파지(close-and-hold)를 못 함 → **0마리 파지·0마리 배치**
  (평가 0/10, 수집분포에서도 0/4).
- 이 원인은 관측 버그(원인 1·2)에 가려 navigation 단계에선 안 보였고, 그것들을 고친 뒤 grasp 단계에서 드러남.

---

## 4. 부차적·구조적 난이도

- **성공 조건 불일치**: BDDL goal은 죽은 병아리 **3마리 전부** 바구니에 담아야 성공인데,
  language instruction은 단수("Pick up the blue chick"). 실제는 **3연속 pick-place**.
- **장기 호라이즌**: 데모 평균 **622 스텝** (단일 pick-place의 약 3배). 평가 max_steps=700.
- **타깃 저대비**: thermal에서 죽은 병아리(파란)가 차가운 파란 바닥과 잘 안 구분됨. 가장 눈에 띄는 건
  오히려 살아있는 따뜻한(밝은) 병아리 → 모델이 골라야 할 대상이 화면에서 가장 덜 보임.

---

## 5. 무엇이 원인이 아니었나 (배제 근거)

| 가설 | 배제 근거 |
|---|---|
| 데이터 양 부족 (100→200) | 모델이 in-distribution 이미지에 5 mm 정밀 → 학습 자체는 성공 |
| 학습 시간 부족 (15k→30k) | 풀 30k 완주 후에도 0% → 미학습 아님 |
| 데이터의 no-op 비율 | no-op 필터링 후에도 동일 실패 |
| 렌더 엔진/env 설정 차이 | 수집·평가 모두 동일 `OffScreenRenderEnv`(256, 동일 컨트롤러) |
| thermal_fx 색 비결정성 | thermal_fx는 결정론적; 해상도만 맞추면 픽셀 단위 동일 |

---

## 6. 결론

본 과제의 0%는 **"학습/데이터 양 실패"가 아니라**:

1. **학습/평가 관측 불일치**(이미지 방향, thermal 해상도, 장면 분포)가 — 잘 학습된 정책을 — 평가 시점에
   무력화시켰고 (정지 → 표류 → 위치추정 8 cm 오차),
2. **거친(bang-bang) 액션 표현**이 마지막 **정밀 파지**(미세 XY·Z 정렬 + 그리퍼 타이밍)를
   학습 불가능하게 만들었다.

관측 불일치들을 바로잡자 모델은 죽은 병아리를 **1 mm까지** 정확히 찾아갔고, 남은 단 하나의 블로커는
**거친 액션으로 인한 파지 실패**다.

## 7. 해결책

1. 컨트롤러 게인 하향(`GAIN_XY 8→~2.5`, `GAIN_Z 6→~2`)으로 **부드러운 액션**(포화 49%→<5%) 데모 재수집.
2. (선택) goal을 1마리로 축소 → 호라이즌 단축, 파지 1회로 요구 완화.
3. RLDS 재빌드 → 재학습.
4. **평가 관측 파이프라인을 수집과 완전히 일치**(랜덤 장면 + wander, 256 thermal, `[::-1]`).

> 시사점: VLA 평가 실패의 상당 부분이 정책 학습 실패가 아니라 **학습/평가 관측·분포 불일치**와
> **액션 표현의 정밀도 한계**에서 온다. 이 진단 자체가 결과로서 의미가 있다.
